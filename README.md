# AirPLAi · Hockey

A domain-expansion demo: AirPLAi's AI sports-video intelligence, applied to **hockey**.

**Live demo:** _Vercel URL (TBD)_ · **Repo:** `github.com/Keegs12/AirPLAi-Sports-NHL-Demo-Project`
· **Part 1 (Domain & Design):** [`docs/DOMAIN_DESIGN.md`](docs/DOMAIN_DESIGN.md)
· **Part 3 (Technical Writeup):** [`docs/TECHNICAL_WRITEUP.md`](docs/TECHNICAL_WRITEUP.md)

The product AirPLAi sells is computer vision over game film. The brief says we don't need to
run CV — public data is enough. So this demo builds the **real product surface** — a
**live game-replay experience** — and feeds it underneath with **public per-game
play-by-play standing in for the CV output**. It looks and behaves like you're watching the
game; under the hood it's public data. Two surfaces public feeds *cannot* provide — the
**goalie net-zone save map** (glove / blocker / five-hole) and **microstats** — are presented
as "CV-generated." That gap is exactly why AirPLAi should exist for hockey.

The hero game is the **2026 Stanley Cup Final, Game 3 (VGK 5, CAR 4, 2OT)** — Marner's
record hat trick, Vegas up 4-0, Carolina's four-goal third-period comeback, a goalie change,
and Theodore's fluke 0.04-xG double-OT winner.

## The experience

Two panes, mirroring AirPLAi's pickleball product IA:

- **Left — the tape.** An **embedded YouTube full-game broadcast** (configurable; falls back to
  a local `public/game3.mp4` then a styled placeholder), with a **score bug + period-by-period
  line score, game clock, and a scrubber with event ticks**. Toggle to a **horizontal 5-on-5
  rink reconstruction** (CV-generated tracking). A **cumulative-xG "pressure" timeline** draws
  the 4-0 wall and the comeback at a glance.
- **Right — the intelligence panel.** Five tabs: **Play by play** (highlight cards + all plays,
  incl. penalties), **Box score** (full ESPN columns, sortable, expandable; plus a **Lines**
  view of the line combinations), **Shot map** (live xG dots with an Ice ↔ Net toggle — Net is
  the goalie save-zone map), **PLAiChat** (natural-language Q&A that *drives the other tabs*),
  and **Analytics** (team-vs-team, momentum, possession, **special teams / power play**, and
  microstats).
- **Cards on click.** A player or goalie opens as a card (never a sixth tab), each with a
  **This game / Season** scope. The flagship CV surfaces live here: goalie **net-zone save
  maps** (*where each goalie got beat*), player **shooting fingerprints**, **average-ice-position
  heatmaps**, and **skating speed** — all badged `AirPLAi CV`, with season-aggregate versions.
- **Connective tissue.** A play, a shot dot, a chat clip, the xG timeline — all seek the
  viewer. That sync is what makes this read as *video intelligence*, not a stats site.

A **/airplai** page makes the differentiator explicit (public box-score vs. AirPLAi CV-only
surfaces), and a **/season** dashboard sits behind the live game via the topbar.

## Run it

```bash
npm install
npm run dev          # http://localhost:3000  (builds seed + enriched game if missing)
```

The broadcast in the left pane is an **embedded YouTube full-game replay** by default (set a
different one with `NEXT_PUBLIC_BROADCAST_YOUTUBE`). With no embed it falls back to a local
**`public/game3.mp4`** and then a styled placeholder, so a clean checkout never breaks.

Optional — upgrade the *season* PLAiChat parser from rules to an LLM (the game PLAiChat is a
self-contained deterministic engine and needs no key):

```bash
cp .env.example .env   # set ANTHROPIC_API_KEY=...
```

## Test it

```bash
npm test          # 46 vitest tests across 7 files
npm run typecheck # tsc --noEmit
```

The suite tests what would *quietly* break the demo, not a token smoke test: timeline math,
replay-at-time-T (score, accumulating shots, goalie change), synthesized-data guards (P1 shots
match the real total exactly, box-score goals reconcile to the final, Andersen 4 GA / Bussi 1
GA, high-danger shots carry higher xG, shooting rate stays hockey-realistic), the
**box-score↔map consistency invariant** (a skater's shot map plots exactly their shots-on-goal;
a goalie's faced map plots exactly their shots-against), and **special-teams** correctness
(PP goals match strength tags, PP opportunities match the penalty events, line combos reference
real roster players).

**CI:** `.github/workflows/ci.yml` runs **typecheck → tests → production build** on every push /
PR, so a clean checkout is proven to build and pass — not just claimed here.

## Data: real where sourced, labeled where modeled

- **Real:** scoring events, final score, rosters, and P1 shot totals (public NHL.com /
  Wikipedia / CBS / CBC recaps) — `data/games/scf-2026-game3.json`.
- **Modeled (deterministic, committed):** non-scoring shot attempts, box-score columns beyond
  G/A/S, the goalie net-zone maps, and the 5-on-5 tracking — generated by
  `scripts/build-game.mjs` using the **shared xG/spatial model** in `scripts/lib/xg-model.mjs`
  (the same model the season seed uses). The enriched file is `…game3.enriched.json`.

```bash
npm run build:game            # rebuild the enriched hero game (deterministic)
npm run seed                  # rebuild the season seed
npm run ingest                # LIVE MoneyPuck season data (run locally; needs moneypuck.com)
npm run ingest:game 2025030314  # LIVE NHL API play-by-play for a real gameId (run locally)
```

The NHL ingest path (`scripts/ingest-game.mjs`) writes the **same schema** the UI already
consumes, so swapping in a real, already-played game needs zero code changes. The hero game
is an authored near-future scenario, so its coordinates/times are representative; the ingest
script is how you'd point the identical UI at exact real data.

## Deploy

Push to Git and import into **Vercel** — zero config. Add `public/game3.mp4` (and optionally
`ANTHROPIC_API_KEY`) in the project if you want film / the LLM season parser. Deploying and
sharing the URL is your call; the repo is ready for it.
